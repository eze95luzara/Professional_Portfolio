## 🧹 Repository Guidance

Para mantener el portafolio limpio y fácil de clonar, conviene omitir carpetas y archivos generados por las herramientas de desarrollo.

### Files and folders to exclude

- Dependencias instaladas: node_modules/, dist/, build/, .angular/, .next/, coverage/
- Entornos virtuales y temporales: venv/, .venv/, **pycache**/, bin/, obj/, target/, .vs/, .idea/, packages/
- Archivos locales de ejecución: logs, backups, bases de datos temporales, capturas o ejecutables no necesarios
- Archivos sensibles o no relevantes: credenciales, configuraciones locales, datos personales o archivos de prueba grandes

### Repository Guidance

To keep the portfolio clean and easy to clone, it is recommended to exclude generated folders and files from development tools.

#### Files and folders to exclude

- Installed dependencies: node_modules/, dist/, build/, .angular/, .next/, coverage/
- Virtual and temporary environments: venv/, .venv/, **pycache**/, bin/, obj/, target/, .vs/, .idea/, packages/
- Local runtime files: logs, backups, temporary databases, screenshots or executables not needed
- Sensitive or irrelevant files: credentials, local configurations, personal data, or large test files
